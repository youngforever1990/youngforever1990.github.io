条件语句

条件语句尽量都加上括号来写以避免错误产生，尽管它可以不需要括号。

优先：

    if (!error) {
        return success;
    }

避免：

    if (!error)
        return success;

    或者

    if (!error) return success;

今年2月，著名的goto fail事件在苹果的SSL/TLS实现中发现了，这个bug的出现是因为在if语句中重复使用了goto 语句，在父类语句中如果包裹if 的话可以避免这个问题产生

代码摘录：

    static OSStatus
SLVerifySignedServerKeyExchange(SSLContext *ctx, bool isRsa, SSLBuffer signedParams,
                             uint8_t *signature, UInt16 signatureLen)
{
        OSStatus        err;
        ...

        if ((err = SSLHashSHA1.update(&hashCtx, &serverRandom)) != 0)
            goto fail;
        if ((err = SSLHashSHA1.update(&hashCtx, &signedParams)) != 0)
            goto fail;
            goto fail;
        if ((err = SSLHashSHA1.final(&hashCtx, &hashOut)) != 0)
            goto fail;
        ...

        fail:
            SSLFreeBuffer(&signedHashes);
            SSLFreeBuffer(&hashCtx);
            return err;
}

很明显，代码中有2个goto fail，我们非常不希望产生类似的bug，另外，这种bug往往与所有的条件语句息息相关，因此也更容易发现；

Php Beginner

php

php 脚本以<?php 开头，以?>

<?php //此处是php代码 ?>

  • $color 用来声明一个变量
  • echo用来输出类似 iOS NSlog printf 输出语句

php变量规则:

  • 变量以$符号开头，以后是变量的名称
  • 变量名称必须以字母或下划线开头
  • 变量名称不能以数字开头
  • 变量名称只能包含字母数字字符和下划线（A-z、0-9 以及 _）
  • 变量名称对大小写敏感（$y 与 $Y 是两个不同的变量）
注释： php变量名称 对大小写敏感

php变量的创建

  • php 没有创建变量的命令，变量会在首次为其赋值时被创建

php变量作用域

  • 函数之外声明的变量拥有 Global 作用域，只能在函数以外进行访问。
  • 函数内部声明的变量拥有 LOCAL 作用域，只能在函数内部进行访问。

global

  • global 关键词用于访问函数内的全局变量。

static

  • 通常，当函数完成/执行后，会删除所有变量。不过，有时我需要不删除某个局部变量。实现这一点需要更进一步的工作。要完成这一点，请在您首次声明变量时使用 static 关键词

echo 和 print 之间的差异

  • echo- 能够输出一个以上的字符串
  • print-只能输出一个字符串，并始终返回1

var_dump()返回变量的数据类型和值
